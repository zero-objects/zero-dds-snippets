# RTI Connext publisher — `@appendable` union (XCDR1)

The publisher is generated by `rtiddsgen -example` from the portable
[`../dyn_app.idl`](../dyn_app.idl) (RTI's generated/licensed source is not vendored
here). Three runtime flags were needed to get a ZeroDDS reader to receive:

```
rtiddsgen -language C++11 -example x64Linux4gcc8.5.0 -unboundedSupport -replace dyn.idl
# in dyn_publisher.cxx: topic "TrackTopic", set the Track fields, and
#   writer_qos << dds::core::policy::Reliability::Reliable();   // see below
make -f makefile_dyn_x64Linux4gcc8.5.0
RTI_LICENSE_FILE=<ndds>/rti_license.dat ./objs/.../dyn_publisher -d 0 -s 400
```

- **License**: set `RTI_LICENSE_FILE` (the runtime, unlike `rtiddsgen`, needs it).
- **Transport**: RTI defaults to SHM + UDP and may deliver data over SHM, which a
  non-RTI reader cannot read. Restrict the participant to UDPv4 in
  `USER_QOS_PROFILES.xml`: `<transport_builtin><mask>UDPv4</mask></transport_builtin>`.
- **Reliability**: the generated example uses a best-effort profile; a reliable
  ZeroDDS reader will not match a best-effort writer for data (only discovery), so
  force the writer reliable.

RTI emits **XCDR1** by default — this leg (with Fast DDS-default) covers the
classic-CDR / `PL_CDR1` reflective-decode path.
